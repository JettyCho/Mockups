# 모니터링 목업 — 요구사항 및 설계 문서

## 1. 목적

- 영업/매출 현황 시각화
- CS/정산 시기 문의 대응 효율화
- 비즈그룹 → 운영그룹 → 캠페인 계층 구조를 한눈에 파악

---

## 2. 데이터 모델

### 2-1. 엔티티 계층 구조

```
비즈니스 그룹 (광고주 or 대행사)
 ├─ 멤버 목록 (초대 완료된 유저만)
 └─ 운영 그룹 (1:N)
      ├─ 승인된 카탈로그 목록
      ├─ 멤버 목록 (타 비즈그룹 유저 교차 등록 가능)
      ├─ 정산 상태
      └─ 캠페인 목록

캠페인
 ├─ 소속: 운영그룹 (→ 비즈그룹)
 └─ 생성자: 유저 (대행사 유저일 수 있음)
```

### 2-2. 비즈니스 그룹

| 속성 | 설명 |
|------|------|
| 이름 | 법인명 |
| 사업자등록번호 | |
| 유형 | `광고주` / `대행사` |
| 멤버 목록 | 이름, 이메일, 연락처 |
| 운영 그룹 목록 | 하위 엔티티 |

### 2-3. 운영 그룹

| 속성 | 설명 |
|------|------|
| 이름 | 브랜드 또는 팀 단위 |
| 승인된 카탈로그 | 브랜드 단위, 보통 운영그룹당 1개 |
| 멤버 목록 | 이름, 이메일, 연락처, **소속 비즈그룹** |
| 정산 상태 | `대상 아님` / `확정 전` / `확정됨` |
| 캠페인 목록 | 캠페인명 + 유형 |

**교차 멤버**: 운영그룹에는 타 비즈그룹(주로 대행사) 소속 유저도 초대할 수 있다. 멤버 조회 시 소속 비즈그룹이 함께 표시된다.

### 2-4. 캠페인

| 속성 | 설명 |
|------|------|
| 캠페인명 | |
| 캠페인 ID | 3자리 숫자 (예: 201) |
| 캠페인 유형 | `협력광고` / `라이브커머스` |
| 캠페인 상태 | `생성됨` → `심사중` → `집행예정` → `집행중` → `집행완료` |
| 기간 | start ~ end date |
| 예산 | 캠페인에 설정된 예산 (원) |
| 소진액 | 실제 소진된 금액 (원) |
| 운영그룹 | 캠페인이 속한 운영그룹 |
| 비즈그룹 | 운영그룹의 상위 비즈그룹 (항상 광고주) |
| 생성자 | 캠페인을 생성한 유저 (이름 + 이메일) |
| 애드그룹 | 연결된 애드그룹 ID (5자리 숫자) |
| SF 기회 | 연결된 세일즈포스 기회 ID (예: 006TJ00000heVnV) |

## 3. 대행사 중심 운영 모델

실제 광고 운영(카탈로그 연동, 캠페인 생성/수정)은 대부분 **대행사 유저**가 수행한다. 이를 반영한 데이터 설계 원칙:

### 3-1. 캠페인 소속과 생성자의 분리

- **협력광고** 캠페인의 `비즈그룹`/`운영그룹`은 **광고주** 소속이다.
- **라이브커머스** 캠페인은 광고주 운영그룹뿐 아니라 **대행사 운영그룹**에서도 운영할 수 있다.
- 캠페인의 `생성자`는 대행사 유저일 수 있다.
- 대행사 유저는 광고주 운영그룹에 **교차 멤버**로 등록되어 캠페인을 생성한다.

### 3-2. 대행사 운영그룹

- 대행사 비즈그룹 하위에도 자체 운영그룹이 존재한다.
- 대행사 운영그룹에서 라이브커머스 캠페인을 직접 운영할 수 있다.
- 협력광고 캠페인은 광고주 운영그룹에만 소속된다.

### 3-3. 예시 데이터 구성

**광고주 4개:**

| 비즈그룹 | 운영그룹 |
|---------|---------|
| (주) 아모레퍼시픽 | 이니스프리, 라네즈, 설화수 |
| (주) LG생활건강 | 더페이스샵, CNP |
| (주) 올리브영 | 올리브영 브랜드관 |

**대행사 4개:**

| 비즈그룹 | 운영그룹 | 담당 캠페인 |
|---------|---------|-----------|
| 매드업 | 매드업 광고운영 | 이니스프리/설화수 캠페인 생성 |
| 주식회사 BAT | BAT 광고운영 | 라네즈/올리브영 캠페인 생성 |
| 씨제이메조미디어 | CJ메조 광고운영 | 더페이스샵/CNP 캠페인 생성 |
| 미핑기획 | 미핑기획 광고운영 | CNP 캠페인 생성 |

**캠페인 10개** (광고주 직접 생성 2개, 대행사 생성 8개):

| ID | 캠페인명 | 생성자 소속 |
|----|---------|-----------|
| 201 | 이니스프리 봄 신상 라이브 | 매드업 |
| 202 | 이니스프리 그린티 협력광고 | **광고주 직접** |
| 203 | 라네즈 수분크림 여름 프로모션 | BAT |
| 204 | 설화수 안티에이징 라이브 | 매드업 |
| 205 | 이니스프리 여름 세일 라이브 | 매드업 |
| 206 | 더페이스샵 여름 선케어 캠페인 | CJ메조 |
| 207 | CNP 프로폴리스 라이브쇼 | CJ메조 |
| 208 | CNP 가을 협력광고 | 미핑기획 |
| 209 | 올리브영 브랜드위크 협력광고 | **광고주 직접** |
| 210 | 올리브영 라이브 뷰티쇼 | BAT |

## 4. 화면 구성

### 4-1. 탭1: 그룹-멤버 조회

#### 필터/검색 바

| 순서 | 필드 | 타입 | 비고 |
|------|------|------|------|
| 1 | 정산 상태 | select | `대상 아님` / `확정 전` / `확정됨` |
| 2 | 비즈그룹/운영그룹 이름 검색 | text input | 부분 일치 검색 |

#### 테이블 컬럼

| # | 컬럼명 | 비고 |
|---|--------|------|
| 1 | 비즈그룹/운영그룹명 | 계층 표시, 아코디언 |
| 2 | 사업자등록번호 | 비즈그룹만 |
| 3 | 유형 | 배지 (비즈그룹만) |
| 4 | 카탈로그 | 태그 스타일 (운영그룹만) |
| 5 | 이번 달 정산 상태 | 배지 (운영그룹만) |
| 6 | 멤버 수 | 칩 스타일, 클릭 → 모달 |
| 7 | 캠페인 수 | 유형별 분리 표시: "라이브커머스 N / 협력광고 M". 각 칩 클릭 → 탭2 해당 서브탭 이동 + 운영그룹 필터 적용 |

#### 동작

- 기본 상태: 모든 비즈그룹이 **펼쳐진 상태** (운영그룹까지 표시)
- 비즈그룹 행 클릭 시 운영그룹 서브행 접기/펼치기
- 멤버 수 칩 클릭 시 멤버 목록 **모달** 표시 (소속 비즈그룹 컬럼 포함, 교차 강조 없음)
- 캠페인 수 칩 클릭 시 탭2로 이동 + 해당 서브탭(라이브커머스/협력광고) 활성화 + 운영그룹 필터 자동 적용

### 4-2. 탭2: 캠페인별 조회

#### 서브탭

| 서브탭 | 기본값 | 비고 |
|--------|--------|------|
| 라이브커머스 | ✅ (기본 활성) | pill 버튼 스타일, 파란 배경 흰 글씨 |
| 협력광고 | | |

서브탭 전환 시 해당 유형 캠페인만 표시하며, 테이블 컬럼 구성이 유형에 맞게 변경된다.

#### 필터/검색 바 (2줄)

**1줄:**

| 순서 | 필드 | 타입 | 비고 |
|------|------|------|------|
| 1 | 상태 | select | `생성됨` / `심사중` / `집행예정` / `집행중` / `집행완료`. **라이브커머스 서브탭에서만 표시** |
| 2 | 비즈그룹 | searchable dropdown | 캠페인 데이터 기반 옵션, 타이핑 필터링, 정확 일치 |
| 3 | 운영그룹 | searchable dropdown | 캠페인 데이터 기반 옵션, 타이핑 필터링, 정확 일치 |
| 4 | 대행사 | searchable dropdown | 캠페인 데이터 기반 옵션, 타이핑 필터링, 정확 일치 |
| 5 | 생성자 이메일 | text input | 부분 일치 검색 |

**2줄:**

| 순서 | 필드 | 타입 | 비고 |
|------|------|------|------|
| 1 | 캠페인명/ID | text input | 전체 너비, 캠페인명 또는 ID 부분 일치 검색 |

| # | 컬럼명 | 비고 |

| 5 | 유니크클릭 | ceil(소진액÷6.7) 값만 표시. 미집행 시 `—` |
| 6 | 비즈그룹 | |

| 9 | 생성자 | 이름 + 이메일 |
| 10 | 애드그룹 | 하이퍼링크 스타일 |
| 11 | SF 기회 | 하이퍼링크 스타일 |

#### 테이블 컬럼 — 협력광고 (10개)

| # | 컬럼명 | 비고 |
|---|--------|------|
| 1 | 캠페인/ID | 캠페인명(볼드 13px) + ID(회색 11px) 2줄 표시, min-width: 240px |

## 5. 색상/스타일 규칙

### 배지

| 대상 | 색상 |
|------|------|
| 광고주 | 파란색 (`#eff6ff` / `#2563eb`) |
| 대행사 | 보라색 (`#f5f3ff` / `#7c3aed`) |
| 정산: 대상 아님 | 회색 |
| 정산: 확정 전 | 노랑 |
| 정산: 확정됨 | 녹색 |
| 상태: 생성됨 | 회색 |
| 상태: 심사중 | 노랑 |
| 상태: 집행예정 | 파랑 |
| 상태: 집행중 | 녹색 |
| 상태: 집행완료 | 진회색 |
| 유형: 협력광고/라이브커머스 | **무채색 통일** (`#f3f4f6` / `#4b5563`) |

### 소진액 프로그레스바

소진 비율에 관계없이 **단일 연한 파란색** (`#60a5fa`). 경고 색상 미사용.

## 6. 공통 UX 원칙

1. **필터 → 검색 순서**: select 타입 필터를 text input 검색보다 앞에 배치한다.
2. **검색 필드 순서 = 테이블 컬럼 순서**: 검색 필드 배치 순서는 테이블 컬럼 순서와 일치시킨다.
3. **바깥 라벨**: 모든 필터/검색 필드에 필드 바깥 상단 라벨을 표시하여, 값 입력 후에도 필드를 식별할 수 있게 한다.
4. **클릭 가능 숫자는 칩 스타일**: 아이콘 + 숫자 조합의 칩으로 표시하며, 넓은 클릭 영역을 확보한다.
5. **기본 펼침**: 탭1에서 운영그룹까지 기본 펼쳐진 상태로 표시한다.
