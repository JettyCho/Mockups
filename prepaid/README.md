# 버즈빌 광고센터 — 비즈머니(선불충전금) 목업

## 프로젝트 개요

버즈빌 광고센터(`ads.buzzvil.com`)에 **선불충전금(비즈머니) 시스템**을 도입하는 프로젝트의 목업 파일입니다.  
현재 이 파일은 **내부 이해관계자 및 경영진 공유용** 유저플로우 목업이며, 실제 개발 스펙 논의의 기반 자료로 활용됩니다.

## 화면 구성 (Screen Map)

상단 `flow-nav` 바에서 화면 간 전환 가능.

| Screen ID | 화면 이름 | 설명 |
|---|---|---|
| `s1` | 비즈머니 메인 | 잔액 카드, 거래 내역 테이블 (전체/충전/차감/환불 탭), 페이지네이션 |
| `s1e` | 비즈머니 메인 (빈 상태) | 잔액 0원, 환불 계좌 미등록 상태 |
| `s2` | 충전 — 금액 입력 | 금액 프리셋, 카드/계좌 결제수단 탭 전환, 스텝 인디케이터 |
| `s3a` | 카드결제 (토스페이먼츠) | 토스페이먼츠 결제창 UI 목업 |
| `s3b` | 계좌입금 안내 | 신한은행 계좌 안내, 입금 완료 CTA + spinner + 성공/실패 결과 |
| `s4` | 충전 완료 영수증 | 영수증 스타일 완료 화면 |
| `s5` | 광고신청 차감 확인 | 부가세 포함 차감 내역, 차감 후 잔액 미리보기 |
| `s6` | 환불 요청 | 환불 금액 입력, 대표 환불 계좌 고정 표시 |

## 핵심 비즈니스 로직 (요구사항 요약)

### 충전
- 최소 충전 금액: **10만 원**, 1만 원 단위
- 충전 방식 2가지:
  - **카드결제**: 토스페이먼츠 PG 연동 (`ads.buzzvil.com` 기준 계약 필요, `dashboard.buzzvil.com`과 별도)
  - **계좌입금**: 신한은행 지정 계좌로 입금 → 최대 24시간 내 확인 후 충전
- 충전금은 **운영그룹** 단위로 관리됨 (비즈니스 그룹과는 다른 개념)

### 입금용 가상계좌 모델
- 입금용 가상계좌는 **운영그룹 단위**로 발급됨 (비즈니스 그룹 단위가 아님)
- 계층 구조: **광고주/비즈니스 그룹** (예: (주) 아모레퍼시픽) > **운영그룹** (예: 이니스프리)
- 대행사 변경 시 **새로운 가상계좌가 발급**됨 (기존 계좌는 시스템에서 유지, UI에서만 새 계좌 노출)
- 입금용 가상계좌 ≠ 환불 계좌 (별개 개념)

### 차감
- 광고 신청/생성 시 **부가세(10%) 포함** 금액이 즉시 차감
- 슬롯형 광고(예: 라이브커머스): 광고 **생성 시점**에 차감
- 소진액 기반 광고(예: 협력광고): 광고 **신청 시점**에 차감, 종료 후 미소진분 환불

### 환불
- 미사용 비즈머니는 언제든 환불 가능 (고객센터 요청)
- 카드 충전분 → 카드 승인 취소
- 계좌 입금분 → 현금 환불 (현재는 수동 처리)
- **무상머니는 환불 불가**
- 환불 계좌는 운영그룹의 **대표 환불 계좌로 고정** (유저가 수정 불가)

### 비즈머니 구조
- **유상머니** (카드/계좌 충전) + **무상머니** (프로모션 지급) 분리 관리
- 소진 순서: 무상머니 먼저 → 유상머니
- 유효기간: 최초 충전일로부터 **최대 3년**, 이후 자동 소멸

### 거래 내역 타입
| 타입 | 설명 |
|---|---|
| 충전 | 카드결제 또는 계좌입금 |
| 차감 | 광고 신청/생성 시 |
| 환불 | 미사용 잔액 환불 |
| 무상 | 프로모션 지급 (보라색 배지) |

## 인터랙션 로직 (JavaScript)

모든 함수는 파일 하단 `<script>` 블록에 위치.

```js
show(id)           // 화면 전환 + flow-nav 활성화 + s3b 상태 초기화
selectPreset(btn)  // 충전 금액 프리셋 선택
selectPayTab(tab)  // 'card' | 'bank' 결제 수단 탭 전환
goToPayment()      // 선택된 결제 수단에 따라 s3a 또는 s3b로 이동
handleBankCTA()    // 계좌입금 "입금 완료" 클릭 → 1.5초 spinner → 성공/실패 번갈아 노출
selectRefundTab(tab) // 'doc' | 'manual' 환불 계좌 등록 방식 전환
```

**`handleBankCTA` 동작 방식**
- 클릭할 때마다 `_bankCTAToggle` 플래그가 토글되어 성공/실패 케이스 번갈아 노출
- 홀수 클릭: ✅ "입금확인이 완료되었습니다"
- 짝수 클릭: ⏱️ "입금확인에 최대 24시간이 소요될 수 있습니다."
- `show('s3b')` 호출 시 상태 초기화됨

## 외부 연동 현황 (개발 참고)

| 항목 | 현황 |
|---|---|
| 토스페이먼츠 계약 | `dashboard.buzzvil.com`은 계약 완료, `ads.buzzvil.com`은 **별도 계약 필요** |
| 신한은행 입금 추적 | API 연동 방식 미정 (신한 Open API / 금결원 오픈뱅킹 / 수동 처리 중 검토) |
| 환불 처리 | 계좌입금분은 현재 **수동 처리** |

## 법적 고려사항

- 비즈머니는 광고 집행용 사이버머니로 **부가가치세 과세 대상 아님** → 세금계산서 발행 대상 아님
- 선불충전금 발행잔액 30억 원 이상 또는 연간 총발행액 500억 원 이상 시 전자금융거래법상 **선불업 등록 의무** 발생 (2024.9.15 개정 시행)
